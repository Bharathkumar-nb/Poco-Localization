Table Exploration

Preliminaries¶

We will start to look at different columns and see how they can help us predict survival. During this module, I'll be building new tables that are a subset of the bigger titanic_table. For instance, perhaps I'll build a table of only female passengers. Or a table with only passengers under the age of 10. Or even a table of female passengers under the age of 10. I'll try to name these tables with something that helps me keep track of what I am doing. I'll have lots of tables floating around (might be a Titanic pun there some place).

Here is the code to set up our table to get started. I add a new line at top that tells the notebook to show what we plot in the notebook. More on plotting later. For now know that you have to put this line as first piece of code in your notebook to see your plots.

Building sub-tables¶

To produce a new table from an existing table calls for a filtered map. We can filter out the passengers we are not interested in. Let's get a start with gender. Let's build a female table. Notice that I am copying over the entire row to the new table - I am using an identity map.

In [2]:
female_table = [row for row in titanic_table if row['Sex'] == 'female']
len(female_table)  # 314 female passengers
Out[2]:
314

Female survivors¶

My next question is how many females survived. In the last module, I did a map to get the Survived column then a reduce using sum to get the count. I'll do that again for practice here but then look at a different way of getting similar results.

One caveat: I am going to combine two map operations into one. In the prior module I used a map to pull out a column and then another map to produce a numeric value from a string. My one map below does both: it pulls out the Survived value and then converts it to an int and then copies it to the new list. If you feel more comfortable doing it as 2 separate maps, no problem.

And note that we are working with the sub-table female_table, not the larger titanic_table.

In [3]:
female_survival = [int(row['Survived']) for row in female_table]
female_survivor_count = sum(female_survival)
female_survival_percentage = float(female_survivor_count)/len(female_table)
print(female_survival_percentage)  # .74 - wow

0.742038216561

So female passengers had twice the survival rate as all passengers taken together.

Getting ready for plotting¶

We could continue to compute percentages and print them out. But sometimes it is more powerful to visualize data in a plot or graph. We are going to need to add a package to Python to do visualization. The package is called matplotlib. The lib is pronounced as in bib. Here is the line of code I will use to pull the package in so I can use it.

import matplotlib.pyplot as plt

You will notice that I actually import from matplotlib.pyplot. Python allows me to choose sub-packages using the dot notation. In essence we are saying look in the (big!) matplotlib package for the sub-package called pyplot - that is the one I want.

The import statement looks a bit odd. We have seen from math import sqrt. I could do something similar here, i.e., import all the functions I will be using. However, it is such standard practice in the python community to use the import as format for matplotlib that I feel loathe to break it.

So what does import as do. It imports all functions in the pyplot sub-package. Similarly, if I did import math I would get all math functions imported. The downside of using import this way is that you have to prefix each function you use with the full package name. I'll give you a couple examples below using math.

In [4]:
from math import sqrt  # what we have seen up until now

print(sqrt(2.7))  # no prefix needed

1.64316767252
In [5]:
import math  # new - pulls in all functions but have to prefix

print( math.sqrt(2.7))  # notice I now need prefix

1.64316767252
In [6]:
import math as my_math  # new - gives nickname to package

print( my_math.sqrt(2.7))  # still have to use nickname

1.64316767252

Getting back to matplotlib, if I wanted to use a function called hist from the package, I would need to write matplotlib.pyplot.hist(). Yuck. To make it more palatable, I can give a nickname to that full path, in this case plt. As you will see, I still need to prefix functions from the package but now get to use plt.hist(). A little better. BTW: the use of plt is also standard. You could choose your own variable nickname but stick with plt to make your life easier.

Plotting female survivors¶

The plotting functions we will use all take one or more lists of values as arguments then plot those values and give us a visualization. We will use our normal way of producing those lists by mapping columns from a table. And we already have a list of survival values in female_survival. Let's try using a histogram (using plt.hist) to see what things look like.

In [7]:
import matplotlib.pyplot as plt

plt.hist(female_survival, 2)  #list of values, number of bins to use

plt.show()

Kind of rough around the edges but not bad. The hist function takes a list of numeric values and a number of "bins". It divides the x axis into the bins and then shows with a bar how many values fall in each bin. We only have 2 bins, 0 and 1.

Let's clean up the plot¶

The problem with the plot above is that I don't know what the visualization means without looking at the code that produced it. I'd like to label the axis and also the plot as a whole. The ticks on the x axis are also odd, being in decimal places. I'll add some code to do clean-up.

Before I get to the code, you need some background on how plotting works. Up until now we have been assigning values to variables. As you can see below, there are no assignment statements. What is happening is that the plt package is working in the background to get our plot ready for us. We give it instructions by calling functions. When we are ready, we tell plt to go ahead and show us what it has. Again, it is a type of pipeline or workflow that is typical of many Python packages.

As you can see, I am putting a label on the x-axis and the y-axis. I am also giving a title to the whole plot. You can choose whatever strings for these that make sense to you.

The last thing I do is to specify that I want the ticks 0 and 1 on the x-axis to replace the default decimal values.

In [8]:
#import matplotlib.pyplot as plt - only need to import once in notebook

plt.hist(female_survival, 2)  # same as before
plt.xlabel("Survival")
plt.ylabel("Female passengers")
plt.title("Female survival")
plt.xticks([0,1])  # a list of ticks

plt.show()  # ok, ready to see what it looks like

Females in 3rd class¶

Let's do one more. I have two tables, one of females only, one of 3rd class only. I'd like to look at the intersection of the two tables: females in 3rd class. Python does have a set intersection function, but we would have to jump through hoops to use it. I'll just use what we already know and build a new sub-table that represents the intersection. Notice I am using the class3_table as a starting point for the first filtered map.
